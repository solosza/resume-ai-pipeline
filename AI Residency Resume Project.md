## Design & Styling Guidelines

### Visual Style
- **Modern, clean, minimal** (not corporate)
- **Developer-focused aesthetic** (think GitHub, not LinkedIn)
- **Dark mode option** (optional but modern)
- **Typography:** Clean sans-serif (Inter, SF Pro, or system fonts)
- **Color scheme:** Subtle, professional (blues/grays, not flashy)
- **Spacing:** Generous whitespace, easy to scan
- **Mobile-first:** Must work perfectly on mobile

### Content Presentation
- **Projects section:** Most prominent, visual hierarchy clear
- **Links:** Obvious and clickable (GitHub repos if available)
- **Sections:** Clear separation, easy navigation
- **Bullet points:** Concise, outcome-focused
- **No walls of text:** Break up content, use spacing

### Technical Considerations
- Single HTML file with embedded CSS
- No external dependencies (for offline viewing)
- Fast load time
- Semantic HTML
- Accessible (proper headings, ARIA where needed)
- Print-friendly CSS (in case they print it)
